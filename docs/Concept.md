# And Planning — AI Operating Systemのコンセプト

---

## AI OSとは何か

**AI Operating System（AI OS）** とは、
複数のAIを統括・管理し、自律的にビジネスタスクを実行するプラットフォームです。

一般的なOSがCPU・メモリ・ストレージを管理するように、
And Planning AI OSは **Claude・ChatGPT・Gemini などの AIリソース** を管理します。

---

## 従来のAI利用との違い

```
【従来の使い方】
人間 → ChatGPTに質問 → 回答を受け取る → 次の質問を考える → ...
※ 人間が毎回指示を出す必要がある
※ 一つのAIしか使えない
※ 結果の管理・活用は人間の手作業

【And Planning AI OS】
人間 → 「記事を書いて」と一言
  → Inbox がタスク受信
  → AI Router が最適なAIを選択
  → Workflow Engine が自動実行
     Step1: Claudeがアイデア生成
     Step2: Claudeが構成作成
     Step3: Claude/ChatGPTが執筆
     Step4: 自動レビュー
     Step5: 人間が承認（30秒）
     Step6: 自動公開・記録・分析
  → Dashboard で全工程確認
※ 人間は承認の30秒だけ介入
```

## AIファクトリーという考え方

And Planningのコアコンセプトは **「AIファクトリー」** です。

```
リアル工場:
  原材料 → [製造ライン] → 完成品

AIファクトリー:
  アイデア → [AIワークフロー] → 成果物（記事・動画・分析）
```

工場と同じように:
- **ライン（Workflow）** は繰り返し実行できる
- **工程（Step）** は最適化・改善できる
- **設備（Factory）** は増設できる
- **品質管理（Review）** は自動化できる

## データフローの原則

### 1. すべてはInboxから始まる
タスクの発生源は問わない。スマホ・PC・スケジューラー・API。
すべてInboxに集約され、均一に処理される。

### 2. AIRouterが最適なAIを選ぶ
タスクの種類・コスト・応答速度に基づいて
Claude・ChatGPT・Geminiを自動選択する。

### 3. Workflowが品質を保証する
一度定義したWorkflowは何度でも同じ品質で実行される。
人間のスキルに依存しない。

### 4. Human Approvalで人間が制御を保つ
重要な判断（公開・送信）は必ず人間が承認する。
AIは自律的に動くが、最終決定権は人間にある。

### 5. Memoryが経験を積み重ねる
過去の実行結果・フィードバックをMemoryに蓄積する。
時間が経つほど、AIの判断精度が上がる。

## ロードマップの哲学

```
Phase 1: まず動く（MVP）
  → 完璧を求めず、実際に動く基盤を作る

Phase 2: 役に立つ（Factory）
  → 実際にビジネス価値を生む成果物を作る

Phase 3: 売れる（Commercial）
  → 他者も使える製品にする

Phase 4: 広がる（Scale）
  → 自動拡張・自律進化するシステムへ
```
